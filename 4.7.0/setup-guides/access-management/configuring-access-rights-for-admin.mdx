Access authorizations are provided, each with specified access scopes:

1. **Manage Platform** - for configuring access to platform management features

Available scopes:

* **read** - Users can view platform status
* **admin** - Users can force health check scans
   
2. **Manage Processes** - for configuring access to process definition management

Available scopes:

* **import** - Users can import process definitions and process stages
* **read** - Users can view process definitions and stages
* **edit** - Users can edit process definitions
* **admin** - Users can publish and delete process definitions, delete stages, and edit sensitive data for process definitions

3. **Manage Configurations** - for configuring access to generic parameter management

Available scopes:

* **import** - Users can import generic parameters
* **read** - Users can view generic parameters
* **edit** - Users can edit generic parameters
* **admin** - Users can delete generic parameters

4. **Manage Users** - for configuring access management permissions

Available scopes:

* **read** - Users can read all users, groups, and roles
* **edit** - Users can create/update any user groups or roles
* **admin** - Users can delete users, groups, or roles

5. **Manage Integrations** - for configuring adapter integrations

Available scopes:

* **import** - Users can import integrations
* **read** - Users can view all integrations, scenarios, and scenario configurations (topics, input model, output model, headers)
* **edit** - Users can create/update/delete values for integrations/scenarios and scenario configurations (topics, input model, output model, headers)
* **admin** - Users can delete integrations/scenarios with all child elements

<Warning>
These roles must be defined in your chosen identity provider solution (such as Keycloak, Red Hat SSO, or another identity provider).
</Warning>

If custom roles are needed, you can configure them using environment variables. Multiple roles can be set for each access scope.

To configure access for each role, use the following format:

**`SECURITY_ACCESSAUTHORIZATIONS_AUTHORIZATIONNAME_SCOPES_SCOPENAME_ROLESALLOWED: NEEDED_ROLE_NAMES`**

Valid values for `AUTHORIZATIONNAME`: `MANAGEPLATFORM`, `MANAGEPROCESSES`, `MANAGECONFIGURATIONS`, `MANAGEUSERS`

Valid values for `SCOPENAME`: `import`, `read`, `edit`, `admin`

For example, to configure role access for read permissions:

```bash
SECURITY_ACCESSAUTHORIZATIONS_MANAGEPROCESSES_SCOPES_READ_ROLESALLOWED: ROLE_NAME_TEST
```
