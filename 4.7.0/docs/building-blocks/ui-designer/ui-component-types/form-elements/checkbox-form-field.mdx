## Overview

Checkbox form fields are essential UI elements that allow users to select one or more options from a predefined set. They are commonly used for:

* Gathering user preferences
* Accepting terms and conditions
* Selecting multiple items from a list
* Enabling/disabling features or options

## Basic Configuration

### Process Data Integration

#### Process Data Key
The process data key creates a direct link between the checkbox element and your process data. This enables you to:

* Store user selections in process variables
* Use checkbox values in [decisions](../../../node/exclusive-gateway-node)
* Leverage data in [business rules](../../../actions/business-rule-action/business-rule-action)
* Pass values to [integrations](../../../node/message-send-received-task-node#from-integration)

<Info>
Choose descriptive process data keys that reflect the purpose of the checkbox field, making it easier to reference the data later in your process.
</Info>

### Core Properties

#### Label Configuration
* **Label Text**: The visible text that appears next to the checkbox
* **Helper Text**: Additional context or instructions shown below the checkbox
  * Can be displayed directly or within an information tooltip

<Frame>
![Checkbox properties configuration](https://s3.eu-west-1.amazonaws.com/docx.flowx.ai/post41/checkbox_props.png)
</Frame>

### Data Source Settings

Configure how the checkbox obtains and manages its data:

* **Default Value**: Initial state of the checkbox (checked/unchecked)
* **Source Type Options**:
  * Static: Manually defined options
  * Enumeration: Options from a predefined enumeration
  * Process Data: Dynamic options from process variables
* **Option Configuration**: Define label-value pairs for each checkbox option

![Checkbox datasource configuration](https://s3.eu-west-1.amazonaws.com/docx.flowx.ai/post41/checkbox_datasource.png)

### Dynamic Behavior Control

#### Hide/Disable Expressions
Control checkbox visibility and interaction using JavaScript expressions:

* **Hide Condition**: Expression that determines when to hide the checkbox
* **Disable Condition**: Expression that controls when the checkbox should be disabled

<Warning>
Always ensure that disabled fields have matching hide expressions configured to maintain consistency across platforms.
</Warning>

### Interactive Behavior

#### UI Actions
Define how the checkbox responds to user interaction:

* **Event Type**: Currently supports `CHANGE` event
* **Action Types**: Various actions to execute when the checkbox state changes
