## Overview

The Select form field is a versatile dropdown component that enables users to choose from a predefined list of options. Each option consists of two parts:

- **Label**: The text displayed to users in the dropdown menu
- **Value**: The data stored in the process when an option is selected

<Info>
For example, if you have an option with label "Sports" and value "S", when a user selects "Sports" from the dropdown, the value "S" will be stored in your process data under the specified process data key.
</Info>

## Configuration Options

### Basic Configuration

The Select element has several core settings that determine its behavior:

<Accordion title="Process Data Key">
The process data key creates a link between the select element and your process data. This enables the selected value to be:

- Used in [decision gateways](../../../node/exclusive-gateway-node)
- Processed in [business rules](../../../actions/business-rule-action/business-rule-action)
- Passed to [integrations](../../../node/message-send-received-task-node#from-integration)
</Accordion>

<Accordion title="Properties">
Core properties that control the select field's appearance and behavior:

- **Label**: Text displayed above the select field
- **Placeholder**: Default text shown when no option is selected
- **Empty Message**: Text shown when no options match the search criteria
- **Search for Options**: Enable/disable option filtering 
- **Has Clear**: Add an option to clear the current selection
- **Helper Text**: Additional context shown below the field (can be hidden in an info point)
</Accordion>

<Tab title="Source Types">
Three ways to populate the select options:

1. **Static**: Manually define fixed options
2. **Enumeration**: Use a predefined enum
3. **Process Data**: Dynamically load options from process data

For static and enumeration types, use the "Add Option" button to define label/value pairs.
</Tab>

### Dynamic Behavior

#### Hide/Disable Expressions

Control the select field's visibility and state using JavaScript expressions:

<Accordion title="Hide Expression">
```javascript
// Example: Hide select when another field is empty
${application.otherField} === null || ${application.otherField} === ""
```
</Accordion>

<Accordion title="Disable Expression">
```javascript
// Example: Disable select based on another field's value
${application.status} === "LOCKED"
```
</Accordion>

<Warning>
When using the disable expression, always configure the same expression in the hide expression path to ensure consistent behavior.
</Warning>

## Advanced Example: Dynamic Dropdowns

Learn how to create dependent dropdowns where the options in one select field depend on the selection in another:

<Steps>
<Step title="Process Setup">
1. Create a task node for data preparation
2. Add a user task node for the form
3. Configure necessary actions
</Step>

<Step title="Data Configuration">
Add a business rule action to prepare the data:

```java
output.put("application",
{
    "client": {
        "identity": [
        {
            "value": "001",
            "label": "Eddard Stark"
        },
        {
            "value": "002",
            "label": "Sansa Stark"
        }
        // Additional options...
    ]},
    "contracts": {
        "001": [
            {
                "value": "c001",
                "label": "Contract 1"
            }
            // Additional contracts...
        ]
    }
});
```
</Step>

<Step title="UI Configuration">
1. Add a card component
2. Configure two select fields:
   - Customer select
   - Contract select (dependent on customer)
3. Add submit button
</Step>
</Steps>
