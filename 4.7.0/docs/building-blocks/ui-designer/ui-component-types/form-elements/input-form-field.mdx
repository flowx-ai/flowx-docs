## Overview

Input form fields are essential UI components that enable users to enter data in your applications. They support various data types, validation rules, and can be customized for different platforms.

## Basic Configuration

### Process Data Key

The process data key creates a binding between the input field and your process data. This enables you to:

- Reference the input value in [decisions](../../../node/exclusive-gateway-node)
- Use the data in [business rules](../../../actions/business-rule-action/business-rule-action)
- Pass values to [integrations](../../../node/message-send-received-task-node#from-integration)

<Info>
Choose descriptive keys that reflect the data's purpose. For example, use `customer.firstName` instead of generic keys like `input1`.
</Info>

### Core Properties

Configure these essential input field properties:

- **Label**: The text displayed above the input field
- **Placeholder**: Helper text shown when the field is empty
- **Type**: The input data type (text, number, email, password)
- **Prefix**: Text or symbol displayed before the input
- **Suffix**: Text or symbol displayed after the input
- **Has Clear**: Enable a button to clear the field's content
- **Helper Text**: Additional guidance, optionally shown in an info tooltip
- **Update on Blur**: Whether to update the value when focus leaves the field

<Tip>
Use clear, concise labels and helpful placeholder text to guide users on what information to enter.
</Tip>

## Dynamic Behavior

### Hide/Disable Expressions

Control input field visibility and state using JavaScript expressions:

#### Hide Conditions
```javascript
// Example: Hide field if income is empty
${application.input.income} === null || ${application.input.income} === ""
```

<Frame>
![Hide Expression Result](https://s3.eu-west-1.amazonaws.com/docx.flowx.ai/release40/input_hide_result.gif)
</Frame>

#### Disable Conditions
```javascript
// Example: Disable when name equals 'TEST'
${application.input.name} == 'TEST'
```

<Warning>
Always configure the same hide expression under both expressions â†’ hide and the disabled condition to maintain consistency.
</Warning>

## Platform-Specific Configuration

### Settings Overrides

Customize input behavior per platform (Web, iOS, Android):

- **Properties**: Label, helper text, placeholder, prefix, suffix
- **Expressions**: Hide conditions
- **Styling**: Platform-specific appearance

Use the import/push feature to copy overrides between platforms:

## Styling Configuration

### Web Styling

Configure web-specific appearance:

#### Icons
- Left and right icons with customizable properties
- Icon selection from Media Library

#### Sizing
- **Fill**: Expands to container width
- **Fixed**: Maintains specific width
- **Auto**: Adjusts to content

### Mobile Styling

Additional options for iOS and Android:

- **Height**: Platform-specific units (pt for iOS, dp for Android)
- Platform-specific style overrides
