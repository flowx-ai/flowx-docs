## Overview

The **Table** component is a powerful web-specific UI element designed for displaying and managing structured data. It integrates seamlessly with FlowX.AI's theming framework to provide a consistent design experience.

<Info>
The Table component is currently available only for web applications. It shares functionality with the [Collection](./collection/collection) component but is optimized specifically for tabular data presentation.
</Info>

## Core Components

### Table Structure

The Table component consists of several key elements:

1. **Table Header (th)**
   - Contains column labels
   - One header cell per column
   - Supports sorting and filtering controls

2. **Table Rows (tr)**
   - Dynamically generated from source data
   - Each row represents one data entry
   - Support for editing and deletion

3. **Table Cells**
   - Hold individual data points
   - Support multiple UI elements (text, images, links, buttons)
   - Configurable styling and layout

4. **Actions Column**
   - Optional column for row-level operations
   - Contains edit and delete icons when enabled
   - Automatically added when row editing/deletion is activated

<Info>
New tables are created with three default columns and corresponding cell placeholders to help you get started quickly.
</Info>

## Configuration Guide

### Basic Setup

#### Source Key Configuration

1. Specify a data source array (e.g., `application.users`)
2. Each array object becomes a table row
3. Properties map to column cells

### Advanced Features

#### Sorting Capabilities

1. **Default Sorting**
   - Enable column sorting
   - Select default sort column
   - Choose sort direction (ascending/descending)

<Check>
Remember to enable the "Sortable" option for your chosen default sort column in cell-level settings.
</Check>

#### Pagination Options

Configure how data is displayed:

- **Page Size**: Set rows per page
- **Navigation**: Page controls
- **Scrollable Mode**: Enable continuous scrolling

## Styling Guidelines

### Table Dimensions

Configure table sizing:

- **Width**: Fill available space (default)
- **Height**: Auto-adjust to content
- **Column Width**: Fixed, Auto, or Fill options

### Cell Styling

Customize cell appearance:

1. **Layout Options**
   - Direction: Horizontal/Vertical
   - Justification: Start/Center/End
   - Alignment: Top/Center/Bottom
   - Spacing: Gap between elements

2. **Column Customization**
   - Width settings
   - Resizable columns
   - Custom styling

## Action Integration

### Table Actions

Configure table-level actions:

1. Set up SaveData action
2. Configure UI action mapping
3. Define custom keys for data updates

### Cell Actions

Implement cell-level actions:

1. **Action Types**
   - Standard actions
   - Process triggers
   - Upload capabilities
   - External links

2. **Configuration**
   - Set table item keys
   - Define action parameters
   - Handle responses

<Warning>
Always configure both Custom Keys and Table Item Save Keys when implementing cell actions to ensure proper data handling.
</Warning>

## Best Practices

1. **Data Management**
   - Keep data sources well-structured
   - Implement proper validation
   - Handle updates efficiently

2. **Performance**
   - Use pagination for large datasets
   - Optimize column configurations
   - Minimize complex cell content

3. **User Experience**
   - Provide clear action feedback
   - Maintain consistent styling
   - Implement intuitive controls

## Troubleshooting

<AccordionGroup>
<Accordion title="Table not displaying data">
Check your source key configuration and ensure data array is properly populated.
</Accordion>

<Accordion title="Sorting not working">
Verify that the sortable option is enabled for the specific column in cell settings.
</Accordion>

<Accordion title="Actions not saving">
Confirm that both Custom Keys and Table Item Save Keys are properly configured.
</Accordion>
</AccordionGroup>

## FAQs

<AccordionGroup>
<Accordion title="Can I nest tables within tables?">
No, nested tables are not currently supported to maintain optimal performance and usability.
</Accordion>

<Accordion title="How do I handle large datasets?">
Use pagination or enable scrollable mode, and implement proper filtering for better performance.
</Accordion>

<Accordion title="Can I customize column widths?">
Yes, through column style options and the user-resizable columns feature.
</Accordion>
</AccordionGroup>
