### Action Edit

- **Name** - Used internally to distinguish between different [actions](../actions/actions) within the process. We recommend establishing a clear naming convention for easy identification.
- **Order** - Determines the execution order when multiple actions exist on the same node.
- **Timer Expression** - Enables a delay using the [ISO 8601 duration format](../node/timer-events/timer-expressions#iso-8601). For example, `PT30S` creates a 30-second delay.
- **Action Type** - Must be set to **Kafka Send Action** for sending messages to external systems.
- **Trigger Type** - Always set to **Automatic**.

<Info>
The Kafka Send Action type is always **Automatic**. These actions automatically trigger when the process reaches their designated step.
</Info>

- **Required Type** - Choose between **Mandatory** or **Optional**. **Automatic** actions are typically set as **Mandatory**, while manual actions can be either.
- **Repeatable** - When enabled, allows the action to be triggered multiple times.
- **Autorun Children** - When enabled, child actions (both mandatory and automatic) execute immediately after the parent action completes.

### Parameters

You can add parameters either through the **Custom** option or by importing predefined parameters from an integration.

<Info>
For detailed information about **Integrations Management**, please refer to [<u>**this documentation**</u>](../../platform-deep-dive/core-extensions/integration-management/).
</Info>

- **Topics** - Specifies the Kafka topics that the external system monitors for requests.
- **Message** - Contains the message payload to be sent.
- **Advanced Configuration (Headers)** - Represents a JSON object sent within the Kafka message headers.

## Kafka Send Action Scenarios

The Kafka Send Action serves as a versatile tool that enables seamless communication across various systems and plugins. It facilitates efficient data transfer, robust document management, notifications, and process initiation.

This action can be applied in numerous scenarios when configuring processes:

- **Communication with External Services**
- **Interaction with Connectors** - For example, integrating a connector in the FlowX.AI Designer ([see example](../../platform-deep-dive/integrations/building-a-connector#integrating-a-connector-in-flowxai-designer))
- **Plugin Integration:**
    - **Document Plugin:**
        - Generate, upload, convert, and split documents ([see examples](../../platform-deep-dive/plugins/custom-plugins/documents-plugin/documents-plugin-overview))
        - Update or delete documents ([see example](../../platform-deep-dive/plugins/custom-plugins/documents-plugin/deleting-a-file))
        - Optical Character Recognition (OCR) integration ([see example](../../platform-deep-dive/plugins/custom-plugins/ocr-plugin#scenario-for-flowxai-generated-documents))
    - **Notification Plugin:**
        - Send notifications ([see example](../../platform-deep-dive/plugins/custom-plugins/notifications-plugin/sending-a-notification))
        - Send emails with attachments ([see example](../../platform-deep-dive/plugins/custom-plugins/notifications-plugin/sending-an-email-with-attachments))
        - One-Time Password (OTP) validation ([see example](../../platform-deep-dive/plugins/custom-plugins/notifications-plugin/sending-a-notification))
        - Forward notifications to external systems ([see example](../../platform-deep-dive/plugins/custom-plugins/notifications-plugin/forwarding-notifications-to-an-external-system))
    - **OCR Plugin**
    - **Customer Management Plugin**
    - **Task Management Plugin:**
        - Perform bulk operations updates ([see example](../../platform-deep-dive/core-extensions/task-management/task-management-overview#bulk-updates))
- **Process Data Operations:**
    - Request data for forwarding or processing
    - Perform data searches ([see documentation](../../platform-deep-dive/core-extensions/search-data-service))
- **Process Initiation:**
    - Start processes via Kafka
    - Utilize process hooks ([see examples](../../flowx-designer/managing-a-process-flow/starting-a-process))

The Kafka Send Action is a powerful tool that enables smooth operations across various aspects of your system, from basic communication to complex document management, notifications, and process control.
