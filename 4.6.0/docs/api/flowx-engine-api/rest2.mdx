---
title: Start Process
api: POST /rtm/api/runtime/app/{appId}/build/{buildId}/process-name/{processDefinitionName}/start
description: This endpoint initiates a process based on the provided process definition name in the specified application runtime environment.
---


## Path Parameters

<ResponseField name="appId" type="string" required>
  The ID of the application.
</ResponseField>

<ResponseField name="buildId" type="string" required>
  The build ID for the runtime environment.
</ResponseField>

<ResponseField name="processDefinitionName" type="string" required>
  The name of the process definition to start.
</ResponseField>

## Headers

<ResponseField name="Authorization" type="string" required={true}>
  Bearer token required for authentication. Format: `Bearer <token>`.
</ResponseField>

<ResponseField name="Content-Type" type="string" required={true}>
  Indicates the media type of the request body. Must be set to `application/json`.
</ResponseField>

<ResponseField name="flowx-platform" type="string" required={true}>
  Identifies the client platform (e.g., `web`).
</ResponseField>

<ResponseField name="Accept" type="string" required={false}>
  Specifies the media types that are acceptable for the response. Example: `application/json, text/plain, */*`.
</ResponseField>

<ResponseField name="Accept-Encoding" type="string" required={false}>
  Indicates supported content encoding algorithms for the response. Example: `gzip, deflate, br, zstd`.
</ResponseField>

<ResponseField name="Accept-Language" type="string" required={false}>
  Specifies language preferences for the response. Example: `en-GB,en-US;q=0.9,en;q=0.8`.
</ResponseField>

<ResponseField name="User-Agent" type="string" required={false}>

</ResponseField>

<ResponseField name="Referer" type="string" required={false}>

## Request Body

This endpoint requires a JSON request body. The exact payload structure should conform to the process definition schema associated with `{processDefinitionName}`.

### Example Request Body

```json
{
  "key1": "value1",
  "key2": "value2"
}
```
