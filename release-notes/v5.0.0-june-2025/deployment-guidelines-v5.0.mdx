<Info>
After upgrading to a new platform version, always ensure that your installed component versions match the versions specified in the release notes. To verify this, navigate to **FlowX.AI Designer > Platform Status**.
</Info>

<Warning title="Breaking Changes in FlowX.AI 5.0">
After upgrading to the 5.0 FlowX.AI release, you cannot import old process definitions or resources from versions older than 5.0.
</Warning>

## Quick Reference

### Current Release Status
- **Production Ready:** FlowX.AI 4.7.4 (December 2024)
- **Technology Preview:** FlowX.AI 5.0 (Q1 2025)

### Version Compatibility


<Warning>
FlowX components must be upgraded together as a complete set. Mixed versions across different releases are not supported and will cause system instability.
</Warning>

| FlowX Release | Compatible Component Range | Upgrade Path |
|---------------|----------------------------|--------------|
| **5.0** | 5.0.x only | Breaking changes - requires migration |
| **4.7.x** | 4.7.0 - 4.7.4 | Patch versions are interchangeable |
| **4.6.x** | 4.6.0 - 4.6.1 | Limited forward compatibility |
| **4.1.x** | 4.1.1 - 4.1.4 | Must upgrade to 4.7.4 first, then to 5.0 |

## Pre-Upgrade Requirements

<Warning title="Action Required: Upgrade Dependencies Before FlowX 5.0">

**Before upgrading to FlowX 5.0, you must upgrade these dependencies:**

| Component | Current Minimum | New Minimum (5.0) | 
| --------- | --------------- | ----------------- | 
| Keycloak  | 22.x            | 26.x              | 
| Kafka     | 3.2.x           | 3.9.x             | 
| Redis     | 7.2.x           | 7.4.x             | 

**Timeline:** Deprecated versions will stop working immediately upon FlowX 5.0 installation.
**Risk:** Attempting to run FlowX 5.0 with older dependency versions will result in system failure.
</Warning>

## Upgrade Process

### Upgrade Sequence

<Warning>
Components must be upgraded in the specified order to prevent dependency conflicts and ensure system stability.
</Warning>

#### Phase 1: Infrastructure (30-45 minutes)
1. **Third-party dependencies** (Keycloak, Kafka, Redis, etc.)
2. **Database migrations** (PostgreSQL/Oracle schema updates)
3. **Elasticsearch indices** (reindex if required)

#### Phase 2: Core Platform (60-90 minutes)
1. **process-engine** (stop all running processes first)
2. **admin** (administration services)
3. **events-gateway** (event processing)
4. **scheduler-core** (task scheduling)

#### Phase 3: Services & Plugins (45-60 minutes)
1. **cms-core** (content management)
2. **audit-core** (auditing services)
3. **All plugins** (notification, document, OCR, task-management, reporting)
4. **data-search** (search functionality)

#### Phase 4: User Interfaces (30-45 minutes)
1. **designer** (process designer)
2. **integration-designer** (integration tools)
3. **application-manager** (application management)

#### Phase 5: Client Components (15-30 minutes)
1. **Mobile renderers** (iOS, Android)
2. **Web SDKs** (Angular, React packages)

#### Phase 6: AI Components (30-45 minutes)
1. **ai-designer** (AI design assistance)
2. **ai-assistant** (user assistance)
3. **ai-analyst** (business intelligence)
4. **ai-developer** (development assistance)
5. **di-platform** (data intelligence)
