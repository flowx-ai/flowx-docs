<Warning>
  After upgrading to the 4.7.x FlowX.AI release, you cannot import old process definitions or resources from versions older than 4.7.0.

Starting with 4.7.0 release, the import-export functionality for application-version is no longer backward compatible.

  * Zip files exported from version 4.7 cannot be imported into earlier FlowX versions.

  * Similarly, zip files from earlier versions cannot be imported into 4.7.

  Note: Zip files for builds and specific resources remain unaffected.
</Warning>

<Info>
  **New in 4.7.5: Extended Support for 3rd Party Dependencies**
  
  FlowX.AI 4.7.5 introduces extended support ranges for third-party dependencies based on their official End-of-Life (EOL) schedules:

  - **Keycloak**: Support from legacy 22 through 26+ with rolling support for future versions
  - **Kafka**: Support from legacy 3.2 through 3.9 (EOL February 19, 2027)
  - **PostgreSQL**: Support from v16 (EOL November 09, 2028) through v17 (EOL November 08, 2029)
  - **Oracle**: Support from 21c (EOL July 31, 2027) through 23ai (EOL December 31, 2031)
  - **MongoDB**: Support from version 7 (EOL August 31, 2026) through 8 (no EOL announced yet)
  - **Redis**: Support from legacy 7.2.x through 8.0 (no EOL announced yet)
  - **Elasticsearch**: Support from version 7 (EOL January 15, 2026) through 9 (no EOL announced yet)

  **Support Timeline**: FlowX.AI 4.7.x LTS support ends January 1st, 2026.

  FlowX.AI supports all versions within the specified ranges of the prerequisite third-party components in the above table.

  For optimal performance and reliability, our internal QA process validates new FlowX.AI releases using the latest stable versions within each supported range. Legacy versions (marked with warning icons) are supported but may have limited testing coverage.

  While exploring alternative versions within the supported ranges, we recommend referring to the table above and considering the EOL dates for long-term planning.

  In the unlikely event that you encounter any compatibility issues with FlowX.AI, please open a support ticket [**here**](https://flowxai.zendesk.com/), and our dedicated team will address and resolve any identified bugs following our standard support process.
</Info>
