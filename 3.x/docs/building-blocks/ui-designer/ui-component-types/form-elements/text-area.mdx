A text area is a multi-line input field that allows users to enter longer pieces of text in forms and conversational interfaces. It's commonly used for:
- Comments and feedback
- Descriptions and detailed explanations
- Multi-line messages
- Notes and observations
- User responses requiring extended text input

#### General
   
* **Process data key** - creates the binding between form element and process data, enabling integration with [decisions](../../../node/exclusive-gateway-node), [business rules](../../../actions/business-rule-action/business-rule-action), and [integrations](../../../node/message-send-received-task-node#from-integration)

#### Properties

* **Label** - display text above the text area
* **Placeholder** - hint text shown inside the empty text area
* **Has clear** - option to add a clear content button
* **Helpertext** - supplementary information about the field (can be displayed in an infopoint)

#### Datasource

Configure default values to pre-populate the text area when the process initializes.

#### Validators

Implement various validation rules for the text area input. See [validation options](../../validators) for detailed configuration.

#### Expressions  

Control the text area's behavior using JavaScript expressions:
   
* **Hide** - JavaScript expression to control visibility
* **Disabled** - JavaScript expression to manage the enabled/disabled state
