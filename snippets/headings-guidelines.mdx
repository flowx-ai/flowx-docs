# Guidelines for Creating LLM-Friendly Headings

## Heading Structure
- Use a clear hierarchy (H1 > H2 > H3 etc.)
- Start with a single H1 title
- Keep heading levels sequential (don't skip levels)

## Heading Content
- Make headings descriptive and meaningful
- Keep headings concise (2-10 words)
- Use natural language, not just keywords
- Include key terms that users might search for

## Formatting Best Practices
- Use title case for main headings
- Use sentence case for subheadings
- Avoid punctuation in headings
- Don't end headings with periods

## Examples

✅ Good heading structure:
```markdown
# Input Form Field Component
## Configuration Options 
### Basic Properties
### Advanced Settings
```

❌ Poor heading structure:
```markdown
# Input Fields
### Properties
## Configuration
#### Settings
```
